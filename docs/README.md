# ドキュメント管理

このディレクトリでは、要件定義や技術検証のためのプロンプト/回答を管理しています。

## ディレクトリ構造

```
docs/
├── YYYYMMDD/              # 日付ごとのディレクトリ
│   ├── 001-prompt.md      # 1プロンプト、1回答のmdファイル
│   ├── 002-prompt.md
│   └── ...
└── README.md              # このファイル
```

## 命名規則

- **ディレクトリ名**: `YYYYMMDD`形式（例: `20260112`）
- **ファイル名**: `NNN-prompt.md`形式（3 桁の連番、例: `001-prompt.md`, `002-prompt.md`）

## ファイル形式

各ファイルは以下の形式で記述します：

```markdown
## プロンプト

[ユーザーの質問や要件]

---

## Answer

[回答内容]
```

## 管理方法

1. 新しいプロンプト/回答を作成する際は、当日の日付ディレクトリを作成（存在しない場合）
2. そのディレクトリ内で連番を付けてファイルを作成
3. 1 プロンプト、1 回答を 1 ファイルにまとめる

## 実装時のドキュメント形式

実装時は、要件定義とは別に簡潔な形式で記録します。

### ファイル名

`implementation-NNN.md` 形式（例: `implementation-001.md`）

### 記録内容

- **実装内容**: 何を実装したか
- **ファイル構成**: 実装した結果、どのような構成でファイルを実装したか
- **エラー**: 実装時にエラーはなかったか
- **懸念点**: 類似実装時に懸念点はないか

### 例

```markdown
## 実装内容

LINE Bot Webhook サーバーの基本実装

## ファイル構成

- `src/webhook_server.py`: Flask ベースの Webhook サーバー
- `src/line_notifier.py`: LINE Messaging API 連携

## エラー

なし

## 懸念点

- 署名検証のテストが不十分。本番前に再確認が必要
```

## 実装時のドキュメント配置

実装時のドキュメント（`implementation-NNN.md`）は、実装を行った日付のディレクトリに配置します。

## 例

```
docs/
├── 20260112/
│   ├── 001-prompt.md      # 初回の要件定義
│   ├── 002-prompt.md      # 実装案の具体化
│   └── implementation-001.md  # 実装記録
└── 20260113/
    ├── 001-prompt.md      # 次の日のプロンプト
    └── implementation-002.md  # 実装記録
```
